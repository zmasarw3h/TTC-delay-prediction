# Data

Raw TTC delay workbooks are not committed to this repository. Place downloaded files under mode-specific folders:

```text
data/raw/
  TTC Bus Delays Data/
    *.xlsx
  TTC Streetcar Delays Data/
    *.xlsx
```

The local demo UI can also use a TTC routes and schedules GTFS zip for route-stop validation:

```text
data/raw/ttc_gtfs.zip
```

You can alternatively point the API at another local GTFS zip with `TTC_GTFS_ZIP_PATH`. GTFS data is used for UI route-stop validation only; model training and prediction still use the normalized `Route` and `Location` text contract.

The cleaner can read `.xlsx`, `.xlsm`, `.xls`, `.xlsb`, and `.csv` files where the required reader engine is installed. Multi-sheet Excel workbooks are combined automatically.

Run the cleaning pipeline from the repository root:

```bash
python -m src.data.clean_data
```

By default, the cleaner looks under `data/raw` and detects common TTC folder names such as `TTC Bus Delays Data` and `TTC Streetcar Delays Data`. You can also run only one mode, or use custom folders, by passing `--bus-raw-dir` or `--streetcar-raw-dir`.

Generated cleaning outputs:

- `data/processed/bus_delays_cleaned.csv`
- `data/processed/streetcar_delays_cleaned.csv`
- `data/processed/ttc_delays_cleaned.csv`

Raw and processed data files are gitignored. Leakage-safe modeling features are generated by `src.features.build_features` from the cleaned data. Weather enrichment is not implemented. `Min Gap` is retained in the cleaned data for auditing but is leakage-sensitive and is excluded from the main model.
